# 智书企飞竞品口径

用途：
- 回答竞品对比
- 给内部业务统一简单口径
- 避免售前表达变成“贬低竞品”或“堆功能清单”

## 总体原则

- 先判断客户关注的是签署、合同全生命周期、AI 审查、法律大模型，还是业务系统。
- 不要直接说“我们全面更强”，要说“如果从某个维度看，我们更适合强调什么”。
- 价格、部署方式、版本权益必须以最新公开资料或商务确认为准。
- 对竞品能力不确定时，明确说“需要以对方当前版本为准”。

## 分类口径

### 电子签 / 合同协同类

代表产品：
- e签宝
- 法大大
- 契约锁

常见强项：
- 电子签、签署流程、合同协同、印章/签署合规能力。
- 部分厂商也会提供 AI 审查、智能起草、内容提取等能力。

智书企飞对比口径：

> 如果客户重点是“签署动作”，电子签厂商会更直接；如果客户重点是飞书内的合同全生命周期管理、法务协同、审批归档、AI 审查和合同数据沉淀，智书AI合同更适合强调“飞书原生 + 合同全链路 + AI Agent”。

### AI 合同审查工具类

代表产品：
- 幂律智能 MeCheck
- 法天使智能审查
- 案牍AI

常见强项：
- 合同条款审查、风险识别、修改建议、审查清单、合同模板或法律知识能力。

智书企飞对比口径：

> 如果客户只需要一个独立审查工具，可以重点比较审查准确率、规则配置、价格和文档体验；如果客户希望审查结果继续进入飞书审批、归档、履约和数据分析，智书AI合同或智审平台更适合强调“审查之后还能继续流转和沉淀”。

### 平台型办公 / 生态产品

代表产品：
- 钉钉智能合同
- 飞书生态内其他合同应用

常见强项：
- 办公平台入口统一，适合已有办公生态的客户。

智书企飞对比口径：

> 如果客户已经深度使用飞书，智书企飞的优势是熟悉飞书生态，并围绕飞书组织、审批、消息、群聊、文档、多维表格做业务化落地；如果客户不在飞书体系内，需要先判断其办公生态和迁移意愿。

### 法律大模型 / API 类

代表产品：
- 通义法睿
- 其他法律大模型 API 或私有化模型服务

常见强项：
- 法律问答、文本生成、文书处理、API 调用、模型能力输出。

智书企飞对比口径：

> 法律大模型更像底层能力或通用法律智能服务；智书AI合同和智审平台更偏业务应用，把 AI 放到合同流程、审查任务、审批归档和客户实际工作台里。客户如果只买 API，需要自己做业务系统和流程落地。

## 建议优先维护字段

每个竞品后续建议按以下字段维护：
- 产品名称
- 提供厂商
- 产品分类
- AI 能力
- 部署方式
- 售卖政策
- 售卖价格
- 适合客户
- 与智书企飞对比口径

## 重要提醒

此前整理过国内合同审查产品、法律大模型、API 价格和应用市场价格，但这些价格变动较快。Skill 回答价格时必须优先查最新资料，不要把历史整理内容当成当前事实。
